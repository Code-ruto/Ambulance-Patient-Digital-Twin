    Event log (newest first · includes crew accept/override audit trail)

      Educational prototype for ITAI 4371 (Digital Twins & Virtual Simulation). Real deployments would use live medical-device feeds, HL7/FHIR data standards, and HIPAA-grade security — represented here in simulated, illustrative form only.

      How to read this app (10-second guide)

      • Colors: green = normal, amber = borderline, red = dangerous. Every color also has a word (✔ ⚠ ⛔) so it works for color-blind viewers.
      • Early-warning score: one number (adapted NEWS2) that combines HR, SpO₂, respiration and blood pressure. ≤4 Low, 5–6 Medium, ≥7 High.
      • Four decisions: the twin recommends equipment, responder level (BLS vs ALS), hospital routing, and triage priority — each with a plain "Why" tied to the inputs.
      • Shock index: HR ÷ systolic BP. Above 0.9 suggests the body is compensating for shock.
      • Solid vs dashed lines: solid = recorded vitals; dashed = where the trend is heading (the forecast).
      • Interventions: click "Give oxygen," "IV fluids," etc. and the patient responds — the twin then recalculates. AI advises; the crew decides.
      • Accept / Override: every recommendation can be accepted or overridden, and every action is written to the audit log.
      • Replay slider: rewind the whole run for a debrief; "Return to live" jumps back.
